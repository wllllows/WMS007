# 仓库管理系统（WMS）— 项目说明文档

## 项目简介

基于 Web 的沙发/家具制造企业仓库管理系统，覆盖原材料采购、车间生产、外协加工、成品入库到销售发货的完整业务流程。

**技术栈**：Python FastAPI + React + TypeScript + Ant Design 5 + MySQL 8.0

**架构**：前后端分离，RESTful API，JWT 认证，四角色权限控制。数据库对象 100% 调用率。

## 二、数据库设计

### 基本信息

- 数据库名：`production_purchase_sales_management`
- 引擎：InnoDB，字符集：utf8mb4
- 连接信息：`root / 123456 / 127.0.0.1:3306`

### 快速创建

首次搭建只需执行一条命令：

```bash
mysql -u root -p123456 < wms-backend/sql/00_master.sql
```

然后填充数据：

```bash
cd wms-backend && source venv/bin/activate && python seed_data.py
```

### 数据依赖关系

```
第1层（先建，无依赖）
    employee, workshop, outsourcing_supplier, payment_status, receipt_status

第2层
    warehouse → employee
    raw_material, finished_product → warehouse
    purchase_order → employee + payment_status
    sales_order → employee
    outsourcing_order → outsourcing_supplier + receipt_status
    transfer_order → workshop

第3层
    work_order_employee → employee + work_order
    work_order_raw_material → work_order + raw_material
    outsourcing_order_raw_material → outsourcing_order + raw_material
    transfer_order_raw_material → transfer_order + raw_material
```

## 七、技术要点

### 认证流程

```
浏览器 → 登录页选角色+输入密码
       → POST /api/auth/login
       → 后端 SHA256 验证
       → 返回 JWT token
       → 前端存 sessionStorage
       → 每次请求自动带 Authorization: Bearer <token>
       → 关闭标签页 = 自动登出
```

### 数据完整性保护（三层）

| 层 | 机制 |
|----|------|
| 数据库 | PRIMARY KEY + FOREIGN KEY + CHECK + 12 触发器 |
| 后端 | Pydantic 校验 + 全局 IntegrityError 处理 + 存储过程事务 |
| 前端 | 表单必填校验 + 操作失败中文提示 + 路由守卫 |

### 前后端通信

```
前端 (localhost:5173) → Axios → 后端 (localhost:8000)
                               → SQLAlchemy ORM / 存储过程
                               → MySQL (127.0.0.1:3306)
                               → 触发器自动执行 → 日志/备份表自动写入
```
