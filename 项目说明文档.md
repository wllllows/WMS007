# 仓库管理系统（WMS）— 项目说明文档

## 项目简介

基于 Web 的沙发/家具制造企业仓库管理系统，覆盖原材料采购、车间生产、外协加工、成品入库到销售发货的完整业务流程。

**技术栈**：Python FastAPI + React + TypeScript + Ant Design 5 + MySQL 8.0

**架构**：前后端分离，RESTful API，JWT 认证，四角色权限控制。

## 二、数据库设计

### 基本信息

- 数据库名：`production_purchase_sales_management`
- 引擎：InnoDB，字符集：utf8mb4
- 连接信息：`root / 123456 / 127.0.0.1:3306`

### 表清单（22 张）

**基础数据（6张）**

| 表名 | 存储内容 |
|------|---------|
| `employee` | 员工（采购员/销售员/仓管/车间负责人） |
| `workshop` | 生产车间 |
| `warehouse` | 仓库（原材料/半成品/成品/外协虚拟/车间现场） |
| `raw_material` | 原料（面料/皮革/五金/木材/辅料） |
| `finished_product` | 成品（沙发/椅子等） |
| `outsourcing_supplier` | 外协厂商 |

**业务单据（7张）**

| 表名 | 存储内容 |
|------|---------|
| `payment_status` | 采购付款单 |
| `receipt_status` | 外协收款单 |
| `purchase_order` | 采购订单 |
| `sales_order` | 销售订单 |
| `work_order` | 生产工单 |
| `outsourcing_order` | 外协订单 |
| `transfer_order` | 调拨单 |

**关联表（5张，多对多）**

| 表名 | 关联 |
|------|------|
| `work_order_employee` | 工单 ↔ 员工 |
| `work_order_raw_material` | 工单 ↔ 原料 |
| `work_order_finished_product` | 工单 ↔ 成品 |
| `outsourcing_order_raw_material` | 外协订单 ↔ 原料 |
| `transfer_order_raw_material` | 调拨单 ↔ 原料 |

**辅助日志（4张）**

| 表名 | 用途 |
|------|------|
| `work_order_log` | 工单操作记录 |
| `work_order_delete_backup` | 工单删除备份 |
| `transfer_order_delete_log` | 调拨删除记录 |
| `material_migration_log` | 物料仓库迁移记录 |

### 数据依赖关系

新增数据时必须遵循依赖顺序：

```
第1层（无依赖）→ 先建
    employee, workshop, outsourcing_supplier, payment_status, receipt_status

第2层（依赖第1层）
    warehouse → employee (负责人)
    raw_material, finished_product → warehouse
    purchase_order → employee + payment_status
    sales_order → employee
    outsourcing_order → outsourcing_supplier + receipt_status
    transfer_order → workshop

第3层（依赖第2层）
    work_order_employee → employee + work_order
    work_order_raw_material → work_order + raw_material
    outsourcing_order_raw_material → outsourcing_order + raw_material
    transfer_order_raw_material → transfer_order + raw_material
```

示例：新增物料时仓库编号必须填 `WH001`~`WH005`。新增采购订单时采购员编号必须填 `EMP001`~`EMP015`。

## 三、后端 API 说明

### 启动方式

```bash
cd wms-backend
source venv/bin/activate          # Windows: venv\Scripts\activate
uvicorn app.main:app --reload --port 8000
```

启动后访问 `http://localhost:8000/docs` 查看 Swagger 在线文档。

### API 模块

| 前缀 | 模块 | 主要端点 |
|------|------|---------|
| `/api/auth` | 认证 | POST `/login` |
| `/api/employees` | 员工 | GET/POST/PUT/DELETE |
| `/api/materials` | 物料 | GET/POST/PUT/DELETE |
| `/api/warehouses` | 仓库 | GET/POST/PUT/DELETE |
| `/api/suppliers` | 供应商 | GET/POST/PUT/DELETE |
| `/api/workshops` | 车间 | GET/POST/PUT/DELETE |
| `/api/purchase-orders` | 采购订单 | GET + POST（调用SP） |
| `/api/sales-orders` | 销售订单 | GET + POST + 付款(SP) |
| `/api/work-orders` | 工单 | GET + 完工(SP) + 领料/退料 |
| `/api/outsourcing-orders` | 外协 | GET + POST + 结算(SP) + 发料/入库 |
| `/api/transfer-orders` | 调拨 | GET + POST(SP) |
| `/api/inventory` | 库存 | GET 查询 |
| `/api/dashboard` | 仪表盘 | GET 统计 |

### 存储过程调用

以下操作调用 MySQL 存储过程，包含事务保护：

| 操作 | 存储过程 | 前端触发 |
|------|---------|---------|
| 创建采购订单 | `sp_create_purchase_order` | 采购入库→新建 |
| 工单完工 | `sp_complete_work_order` | 工单管理→完工按钮 |
| 销售付款 | `sp_record_sales_payment` | 销售出库→付款按钮 |
| 调拨物料 | `sp_transfer_material` | 调拨管理→新建 |
| 外协结算 | `sp_finish_outsourcing_order` | 外协订单→结算按钮 |

### 全局异常处理

后端注册了两个全局异常处理器：

- `IntegrityError`：外键冲突返回"关联数据不存在"，重复编号返回"数据已存在"
- `Exception`：兜底捕获，返回具体错误原因

## 五、触发器说明

| 触发器 | 表 | 触发时机 | 作用 |
|--------|-----|---------|------|
| `trg_purchase_order_after_insert` | purchase_order | INSERT后 | 自动累加付款单待付金额 |

| `trg_outsourcing_order_before_update` | outsourcing_order | UPDATE前 | 金额负数/状态回退时拒绝 |

## 七、技术要点

### 认证流程

```
浏览器 → 登录页选角色+输入密码
       → POST /api/auth/login
       → 后端 SHA256 验证密码
       → 返回 JWT token
       → 前端存 sessionStorage
       → 每次请求自动带 Authorization: Bearer <token>
       → 关闭标签页 = 自动登出
```

### 数据完整性保护（三层）

| 层 | 机制 |
|----|------|
| 数据库层 | PRIMARY KEY + FOREIGN KEY + CHECK + 触发器拦截 |
| 后端层 | Pydantic 校验 + 全局 IntegrityError 处理 |
| 前端层 | 表单必填校验 + 操作失败中文提示 |

### 前后端通信

```
前端 (localhost:5173) → Axios HTTP 请求 → 后端 (localhost:8000)
                                          → SQLAlchemy ORM / 存储过程
                                          → MySQL (127.0.0.1:3306)
                                          → 触发器自动执行
```
